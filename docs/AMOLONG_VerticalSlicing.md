# AMOLONG_VerticalSlicing

This file is the Markdown version of `AMOLONG_VerticalSlicing.pdf`.

Vertical slicing architecture organizes a project by feature or screen instead of grouping files only by technical layer. In a vertically sliced project, each screen contains the files that are directly related to that feature, such as the Activity, Presenter, Contract, and any nearby logic that belongs to that screen. In Renvest, this means the selected screens are separated into `login`, `register`, and `dashboard` packages, while shared data classes remain in the `data` package. This structure keeps the screen implementation easy to follow because the View and Presenter for one feature are located together.

This approach is useful because it improves readability, maintenance, and feature isolation. When a change is needed in the login screen, the developer can open the `login` package and immediately find the files responsible for that feature. The same applies to registration and dashboard behavior. Vertical slicing also reduces confusion in larger projects because one feature does not get spread across unrelated directories. Shared models or repositories are still allowed, but they do not remove the clear boundaries between features. For this project, vertical slicing makes the implementation more organized and also makes the architecture easier to explain in documentation and presentation.
